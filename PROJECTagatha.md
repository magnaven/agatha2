## What Agatha is

Agatha is an open source web app that helps women investigate their own health. The core idea: a woman forms a hypothesis, runs it as a structured investigation over time, and shares her findings with others. It is a **citizen science platform for women's health** — not a symptom tracker, not a health dashboard.

The central metaphor is **the investigation**. Every user is a detective building a case file. Journal entries, blood markers, research papers, and community findings are all evidence pinned to a timeline. Agatha's job is to help connect the evidence into a story the user can act on.

**The key differentiator**: Agatha lets users turn a personal investigation into a **shared experiment** that other women can join with their own data. At the end, Agatha drafts a collective finding. This is real-world women's health data that no clinical trial has ever captured.

## Database schema (Supabase)

```sql
-- Core user profile (extended from Supabase auth.users)
profiles (
  id uuid references auth.users primary key,
  name text,
  age_range text,                          -- 'under-25' | '25-29' | '30-34' | '35-39' | '40+'
  onboarding_selections text[],            -- ['symptoms', 'ttc', 'hashimotos', ...]
  conditions text[],                       -- diagnosed conditions from screener
  suspected_conditions text[],            -- 'no-diagnosis-but-suspect'
  investigation_question text,             -- their first investigation in plain language
  muted_tests text[],                      -- blood test reminder mutes
  created_at timestamptz default now()
)

-- Investigations — each user has one active investigation (extensible to multiple)
investigations (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references profiles,
  title text,                              -- auto-generated by Agatha from onboarding
  hypothesis text,
  conditions text[],
  interventions text[],
  started_at timestamptz default now(),
  status text default 'active',            -- 'active' | 'complete' | 'paused'
  experiment_id uuid references experiments -- null if solo, set if part of shared experiment
)

-- Journal / timeline entries (voice notes, written notes, milestones)
journal_entries (
  id uuid primary key default gen_random_uuid(),
  investigation_id uuid references investigations,
  user_id uuid references profiles,
  entry_type text,                         -- 'voice' | 'written' | 'milestone'
  raw_text text,                           -- transcribed voice or written content
  extracted_tags jsonb,                    -- {energy: 'up', brain_fog: 'down', day: 18}
  day_number integer,                      -- day N of investigation
  created_at timestamptz default now()
)

-- Blood markers
blood_markers (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references profiles,
  marker_name text,                        -- 'TPO Antibodies' | 'TSH' | 'Fasting Insulin' etc
  value numeric,
  unit text,
  tested_at date,
  source text default 'manual',            -- 'manual' | 'apple-health' (future)
  created_at timestamptz default now()
)

-- Research cards (pinned to timeline from PubMed or curated)
research_cards (
  id uuid primary key default gen_random_uuid(),
  investigation_id uuid references investigations,
  user_id uuid references profiles,
  source text,                             -- 'curated' | 'pubmed' | 'clinicaltrials'
  title text,
  abstract_snippet text,
  pubmed_id text,
  study_type text,                         -- 'RCT' | 'systematic-review' | 'observational'
  year integer,
  tags text[],
  pinned_at timestamptz default now()
)

-- Shared experiments
experiments (
  id uuid primary key default gen_random_uuid(),
  creator_id uuid references profiles,
  title text,
  hypothesis text,
  protocol jsonb,                          -- {track: [...], duration_days: 90, baseline_markers: [...]}
  conditions text[],
  interventions text[],
  status text default 'active',            -- 'active' | 'complete' | 'draft'
  participant_count integer default 0,
  started_at timestamptz default now(),
  concluded_at timestamptz,
  agatha_draft_conclusion text,            -- auto-generated
  published_conclusion text               -- creator-edited and published
)

-- Experiment participants (each has their own linked investigation)
experiment_participants (
  experiment_id uuid references experiments,
  user_id uuid references profiles,
  investigation_id uuid references investigations,
  joined_at timestamptz default now(),
  primary key (experiment_id, user_id)
)
```

**Row Level Security**: Every table must have RLS enabled. Users can only read/write their own rows. Experiment participant updates are readable by other participants (anonymised).

## Key feature specs

### 1. Branching onboarding screener

**Entry question**: "So, what brings you here?" — 9 multi-select options.

**Branches** (detected from selection combination):
- `curious` → 1 question about interest, light path
- `symptoms` → open text description + diagnosis status
- `fertility` → age range → timing question (age-conditional) → conditions
- `menopause` → symptom multi-select + HRT status
- `complex` (fertility + menopause combined) → priority question

**Conditions question** (fertility + symptoms branches):
"Have you ever been diagnosed with any of the following?"
Multi-select from: Adenomyosis, Asherman's Syndrome, Endometriosis, Endometrial hyperplasia, Fallopian tube blockage, Fibroids, Hypothalamic Amenorrhea, PID, PCOS, Polycystic ovaries (no PCOS diagnosis), POI, Menopause/Perimenopause, RED-S, Uterine polyps, No diagnosis but suspect something, No I haven't.

Selecting certain conditions triggers follow-up questions (Endometriosis → stage, PCOS → diagnostic criteria + features, POI → age at diagnosis, Fibroids → location, HA → period status, RED-S → specialist, Adenomyosis → diagnosis method).

"No diagnosis but suspect something" can combine with conditions.
"No I haven't" is exclusive.

**Output**: Synthesis screen with investigation title (auto-generated by Claude) + first timeline entry created.

### 2. Voice journaling

1. User taps "Voice note" FAB
2. Web Speech API records + transcribes
3. Transcript sent to Claude API: extract conditions, interventions, symptoms, sentiment, day markers
4. Tags displayed as pills below entry
5. Entry pinned to timeline

### 3. Blood test recommendation engine

**File**: `src/lib/bloodTestRecommendations.ts`

Returns `TestRecommendation[]` based on: user profile (conditions, age range), logged markers, active experiments, recent journal entries.

**Tier 1** (every user): Full blood count, Vit D (target 50-80 ng/mL), B12, Ferritin (target 50-150), HbA1c (target <5.4%), Fasting glucose (target <90 mg/dL), Cholesterol panel, Triglycerides (target <100 mg/dL).

**Tier 2** (Attia framework, added for 30s+, all conditions): Fasting insulin (target <6 uIU/mL), Uric acid (target 2-3, flag >5), Homocysteine (target <10 μmol/L), ALT/AST (target <20 U/L), hsCRP (target <1.0 mg/L), ApoB.

**Calculated markers** (auto-derived, no test needed):
- HOMA-IR = (fasting insulin × fasting glucose) / 405, target <1.0, flag >1.5
- Trig/HDL ratio = triglycerides ÷ HDL, target <1.5, flag >2.0

**Condition additions**:
- Hashimoto's: TSH (90d, target 0.5-2.0), Free T3/T4 (90d), TPO Abs (180d), Reverse T3 (180d) + always add Tier 2
- PCOS: Free Testosterone (180d), SHBG (180d), LH/FSH ratio (180d), Fasting insulin HIGH PRIORITY (90d), DHEA-S (365d)
- Endometriosis: CA-125 (365d), hsCRP (180d), Vit D (90d — strongly linked to severity)
- Autoimmune: ANA (365d), hsCRP (180d), ESR (180d), Complement C3/C4 (365d)
- Perimenopause: Estradiol (90d), FSH (90d), Progesterone day-21 (90d), AMH (365d) + full Tier 2
- Fertility planning: AMH (365d), FSH day-3 (180d), Estradiol day-3 (180d), Fasting insulin

**Age additions**: 40s+ → ApoB, Lp(a) one-time, coronary calcium score recommendation. 50s+ → bone turnover markers.

**Journal pattern detection** (scan last 14 entries):
- "energy crash" / "tired after eating" / "afternoon slump" → suggest fasting insulin + CGM
- "brain fog" / "can't concentrate" + fatigue → suggest thyroid panel + homocysteine
- "sugar craving" / "need sugar" → suggest HbA1c + fasting insulin + uric acid

**Disclaimer** (shown on /markers/reminders): "Agatha's targets are based on optimisation research, not just disease prevention. They are often tighter than standard lab reference ranges. Always discuss results and testing frequency with your doctor."

### 4. Overnight research processing

**Trigger**: Vercel Cron job, runs at 02:00 UTC daily.

For each active investigation with a new journal entry in the last 24h:
1. Extract condition + intervention from recent entries (Claude API)
2. Query PubMed NCBI API for matching studies (filter: women's health, last 10 years, RCTs + systematic reviews first)
3. Check curated research table for manually reviewed studies on same topic
4. Create `research_cards` record pinned to today on the timeline
5. Card appears next morning labelled "Agatha · overnight research"

PubMed API endpoint: `https://eutils.ncbi.nlm.nih.gov/entrez/eutils/` (no API key required, rate limit: 3 req/sec)

### 5. Shared experiments

**Create**: Title, hypothesis, protocol (what to track, duration, baseline markers required), conditions filter.

**Join**: User signs up → Agatha creates a linked investigation using the experiment protocol as starting template → user runs their own investigation within the shared frame.

**Participant data**: Each participant's data stays in their own investigation. Aggregate stats (% with improvement, median change) shown to all participants.

**Conclusion**: At experiment end, Claude API drafts collective finding from all participants' anonymised data. Creator edits and publishes. Published conclusion visible on the experiment card in Discover.

## Important constraints

- **Never** use 1px borders for section separation — tonal background shifts only
- **Never** hardcode Anthropic API key — it's injected by Vercel environment
- **Always** use Supabase RLS — health data is sensitive
- **Always** show the medical disclaimer on blood test recommendations
- **Always** label PubMed results as "auto-matched" and curated results as "Agatha reviewed"
- **Always** label ClinicalTrials links as opening externally — Agatha never manages applications
- **Never** reproduce or summarise more than a snippet of a study abstract — link to PubMed
- PubMed API: cache results aggressively, rate limit is 3 req/sec
- Voice transcription: test on mobile Safari early — Web Speech API support varies
